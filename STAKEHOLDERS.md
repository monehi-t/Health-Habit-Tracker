# Stakeholder Analysis – Health Habit-Tracker

## Overview

This document identifies the key stakeholders of the Health Habit-Tracker system, their concerns, pain points, and identifies their success metrics.

---

## Stakeholder Table

| Stakeholder             | Role                                           | Key Concerns                                           | Pain Points                                | Success Metrics                                          |
| ----------------------- | ---------------------------------------------- | ------------------------------------------------------ | ------------------------------------------ | -------------------------------------------------------- |
| Fitness User            | Primary user who tracks workouts and nutrition | Easy tracking, personalized plans, progress visibility | Using multiple apps, inconsistent tracking | 90% daily log completion rate, improved goal achievement |
| Beginner User           | New to fitness and needs guidance              | Simplicity, guidance, ease of use                      | Overwhelmed by complex apps                | Can create first plan in <5 minutes                      |
| Fitness Coach           | Provides workout and nutrition guidance        | Ability to monitor users, track progress               | Lack of centralized client data            | Track ≥10 clients efficiently                            |
| System Administrator    | Manages system operations                      | System uptime, user management                         | Manual monitoring, system crashes          | 99% uptime, quick issue resolution                       |
| Developer               | Maintains and updates system                   | Clean architecture, scalability                        | Poor documentation, tight coupling         | Faster feature delivery, low bug rate                    |
| Nutrition Data Provider | External API provider                          | Proper API usage, data accuracy                        | Excessive API requests                     | Stable API usage within limits                           |
| Business Owner          | Oversees product success                       | User growth, engagement, retention                     | Low engagement, churn                      | 30% increase in active users                             |
